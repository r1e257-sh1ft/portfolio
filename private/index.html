プロフィール

IT業界約9年。ヘルプデスク、システム運用保守、運用改善・自動化に従事してきました。 単なる運用対応に留まらず、課題分析から自動化・標準化・可視化によって再利用可能な仕組みへ改善することを強みとしています。

リーダー経験

5名体制のヘルプデスクチームリーダーとして、問い合わせ対応、FAQ・手順書整備、メンバー育成、対応品質改善を推進しました。

PMO経験

プロジェクトマネジメント支援として、進捗管理・課題管理・品質管理・会議運営を担当。 成果物レビュー運用の再設計や品質基準の標準化を主導し、不備件数30〜40％削減、レビュー工数20％削減を実現しました。

プロジェクト

運用品質向上と業務効率化を目的とした改善プロジェクトを、改善前 / 改善後で整理しています。

監視アラーム最適化

改善前: 不要アラームが大量発生し、重要障害への対応遅延リスクが存在。

改善後: アラーム件数約97％削減、対応工数約96％削減。

使用技術: Excel分析 / Zabbix / Nagios

障害対応業務の自動化

改善前: 復旧作業が担当者依存で、品質ばらつきと確認漏れリスク。

改善後: 工数約23％削減、ヒューマンエラー防止、標準化された復旧手順。

使用技術: Tera Term Macro / スクリプト

生成AI活用による障害判断支援

改善前: 速報発報判断に時間がかかり、担当によって判断基準がばらつく。

改善後: 発報判断時間を大幅短縮し、判断品質を均一化。

使用技術: Power Automate / Copilot Studio / Teams

PMOによるレビュー品質改善

改善前: 成果物レビューが属人化し、不備が多発していた。

改善後: 不備件数30〜40％削減、レビュー工数20％削減。品質基準を標準化し、チーム全体の品質を向上。

役割: PMOサブリーダー（15名規模） / 使用技術: チェックリスト標準化 / バージョン管理 / 進捗管理
